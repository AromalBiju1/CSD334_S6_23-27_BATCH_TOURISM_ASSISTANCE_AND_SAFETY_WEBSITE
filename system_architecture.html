Complete Data Flow Diagram

Authentication Flow

sequenceDiagram
    participant U as 🧑 User
    participant FE as ⚛️ React
    participant API as ⚡ FastAPI
    participant DB as 💾 SQLite

    U->>FE: Enter email + password
    FE->>API: POST /api/auth/login
    API->>DB: Query User by email
    DB-->>API: User record
    API->>API: Verify password (Argon2)
    API->>API: Sign JWT (RS256 private key)
    API-->>FE: access_token + token_type
    FE->>FE: Store token in localStorage

City Exploration Flow

sequenceDiagram
    participant U as 🧑 User
    participant FE as ⚛️ React
    participant API as ⚡ FastAPI
    participant DB as 💾 SQLite

    U->>FE: Navigate to /explore
    FE->>API: GET /api/cities?limit=1000
    API->>DB: SELECT * FROM cities
    DB-->>API: City list with safety zones
    API-->>FE: CityResponse[]
    FE->>FE: Render SafetyMap (Leaflet)
    Note right of FE: 🟢 Green = Safe
    Note right of FE: 🟠 Orange = Moderate
    Note right of FE: 🔴 Red = High Risk

Smart Route Finding Flow (OSRM + Safety Agent)

sequenceDiagram
    participant U as 🧑 User
    participant FE as ⚛️ React
    participant API as ⚡ FastAPI
    participant DB as 💾 SQLite
    participant OSRM as 🗺️ OSRM
    participant SA as 🛡️ Safety Agent

    U->>FE: Enter origin + destination
    FE->>API: GET /api/routes/smart
    API->>DB: Resolve city names to lat/lng
    API->>OSRM: GET /route/v1/driving/{coords}
    Note right of OSRM: Request up to 3 alternatives
    OSRM-->>API: Encoded polylines + distance + duration
    API->>API: Decode polylines to coordinates
    API->>DB: Load all cities for safety data
    API->>SA: evaluate_routes(osrm_routes, cities)
    SA->>SA: Sample 25 points per route
    SA->>SA: Map points to nearest city zones
    SA->>SA: Compute safety score (0−100)
    SA->>SA: Composite score = 60% safety + 20% distance + 20% duration
    SA->>SA: Rank routes + generate reasoning
    SA-->>API: Scored routes + recommended index
    API-->>FE: SmartMultiRouteResponse
    FE->>FE: Render routes with safety overlays

Live GPS Safety Check & Auto-Rerouting

sequenceDiagram
    participant U as 📍 GPS
    participant FE as ⚛️ React
    participant API as ⚡ FastAPI
    participant DB as 💾 SQLite
    participant OSRM as 🗺️ OSRM
    participant SA as 🛡️ Safety Agent

    U->>FE: Position update during navigation
    FE->>API: GET /api/routes/check-safety?lat&lng
    API->>DB: Load all cities
    API->>SA: find_nearest_city(lat, lng)
    SA-->>API: Nearest city + safety zone

    alt 🔴 Zone = RED (DANGER)
        API-->>FE: trigger_reroute = true
        FE->>API: GET /api/routes/reroute
        API->>OSRM: Fresh routes from current pos
        API->>SA: Re-score new routes
        API-->>FE: New SmartMultiRouteResponse
        FE->>FE: Update navigation with safer route
    else 🟢🟠 Zone = GREEN or ORANGE
        API-->>FE: safe = true/false, stay on course
    end

A* Fallback Flow (when OSRM is unavailable)

sequenceDiagram
    participant API as ⚡ FastAPI
    participant DB as 💾 SQLite
    participant OSRM as 🗺️ OSRM

    API->>OSRM: Request routes
    OSRM--xAPI: ❌ Timeout / Error

    Note over API: Fallback to A* graph search
    API->>DB: Load all cities
    API->>API: build_graph(cities, weights)
    Note right of API: 3 weight profiles
    Note right of API: Safest: red = 10x penalty
    Note right of API: Balanced: red = 2x penalty
    Note right of API: Fastest: red = 1.2x penalty
    API->>API: A* search with safety-weighted edges
    API->>API: Calculate safety scores
    API-->>API: MultiRouteResponse with fallback warning

Data Processing pandas + numpy Feature engineering from NCRB data
